## 1) What eCommerce SEO Optimizes

eCommerce SEO focuses on three page types with distinct intent:

* **Transactional pages** (Products + Collections) ‚Äî High purchase intent
* **Informational pages** (Blog + Guides) ‚Äî Top-of-funnel awareness
* **Navigational pages** (Homepage + Categories) ‚Äî Brand + category discovery

Google is currently prioritizing eCommerce stores in search. The key is matching page type to search intent.

## 6) eCommerce SEO Audit Checklist (80+ Points)

**Goal:** Establish baseline trust and tracking.

- [ ] Connect Google Search Console to see search appearance and alerts
- [ ] Connect Google Analytics to Search Console for detailed performance data
- [ ] Create trust pages: About Us, Contact, Privacy Policy, Terms of Service
- [ ] Ensure HTTPS with valid SSL certificate
- [ ] Verify site is mobile-friendly
- [ ] Check robots.txt isn't blocking important pages
- [ ] Submit XML sitemap to Search Console

**Priority:** P0 ‚Äî These block everything else if broken.

### B) Competitor Research / Content Planning

**Goal:** Identify keyword opportunities and content gaps.

**Keyword Research**

- [ ] Conduct keyword research for product categories
- [ ] Identify collection page keyword opportunities
- [ ] Research informational/blog keyword opportunities
- [ ] Analyze competitor keyword strategies
- [ ] Map keywords to user intent (transactional vs informational)

**Content Strategy**

- [ ] Plan content calendar in advance
- [ ] Align keywords with correct page types
- [ ] Create BOFU content (bottom-of-funnel, buying intent)
- [ ] Create TOFU content (top-of-funnel, awareness + email capture)
- [ ] Identify content cannibalization issues
- [ ] Check for duplicate content across site
- [ ] Plan hub-and-spoke content architecture
- [ ] Schedule regular content updates/refreshes

**Content Optimization**

- [ ] Write for users first, search engines second
- [ ] Include target keyword in URL, title, H1

- [ ] Add alt text to all images
- [ ] Link to relevant categories, products, and articles

### C) Product Page SEO

**URL + Structure**

**Content**

### G) Technical SEO

**Goal:** Ensure search engines can crawl and index correctly.

- [ ] Create and submit XML sitemap

- [ ] Don't block CSS/JS files

**Canonicals + Duplicates**

- [ ] Set canonical tags on all pages
- [ ] Handle product variants correctly (canonical to main)
- [ ] Handle pagination with rel="next/prev" or view-all
- [ ] Handle filtered URLs (noindex or canonical)

**Redirects + Links**

- [ ] Fix broken internal links
- [ ] Fix broken external links
- [ ] Resolve redirect chains (max 1 hop)

## 9) Platform-Specific Notes

### Shopify

* Speed is mostly handled ‚Äî focus on image optimization + minimal apps
* Use a fast theme (Dawn, Prestige, Impulse)
* URL structure is fixed ‚Äî work within it
* Use Shopify's built-in blog
* Collection filtering can create duplicate URLs ‚Äî handle with robots.txt or theme settings

### WooCommerce

* More control but more responsibility
* Use caching plugin (WP Rocket, LiteSpeed)
* Use SEO plugin (Yoast, RankMath)
* Watch for plugin bloat

### Magento

* Powerful but complex
* Requires technical resources
* Built-in SEO features are solid
* Performance requires proper hosting + optimization

### BigCommerce

* Good built-in SEO features
* Similar constraints to Shopify
* URL structure is customizable

## 10) Decision Trees

### Page Type ‚Üí Keyword Intent

```
IF intent == "buy [product]" or "[product] price":
  ‚Üí Product page

IF intent == "[category] for [modifier]" or "best [category]":
  ‚Üí Collection page (if product listing)
  ‚Üí Blog post (if comparison/guide)

IF intent == "how to [topic]" or "what is [topic]":
  ‚Üí Blog post / Guide

IF intent == "[brand name]":
  ‚Üí Homepage
```

### Duplicate Content Handling

```
IF product variants (color, size):
  ‚Üí Canonical to main product
  ‚Üí OR use variant schema

IF filtered collection URLs:
  ‚Üí Option 1: Noindex filtered pages
  ‚Üí Option 2: Canonical to main collection
  ‚Üí Option 3: Block in robots.txt

IF paginated collections:
  ‚Üí Ensure all pages are crawlable
  ‚Üí OR implement view-all page with canonical
```

### Out-of-Stock Products

```
IF temporarily out of stock:
  ‚Üí Keep page live
  ‚Üí Add "notify me" email capture
  ‚Üí Update availability schema

IF permanently discontinued:
  ‚Üí 301 redirect to:
    ‚Üí Similar product (if exists)
    ‚Üí Parent collection (if no similar)
    ‚Üí Homepage (last resort)
```

## 11) Error Handling

```
IF site returns 5xx errors:
  ‚Üí P0 Issue: "Server errors blocking crawling"
  ‚Üí Fix: Check hosting, server logs

IF key pages return 404:
  ‚Üí P0 Issue: "Important pages not found"
  ‚Üí Fix: Restore or redirect

IF robots.txt blocks /collections/ or /products/:
  ‚Üí P0 Issue: "Robots.txt blocking indexation"
  ‚Üí Fix: Update robots.txt

IF no sitemap found:
  ‚Üí P1 Issue: "XML sitemap missing"
  ‚Üí Fix: Generate and submit

IF schema validation fails:
  ‚Üí P2 Issue: "Invalid structured data"
  ‚Üí Fix: Correct schema markup
```

## 14) Keyword Cannibalization Detection

**Method 1: Site Search**
```
site:[domain] "[target keyword]"
```
Multiple pages ranking for same keyword with same intent = Cannibalization

**Common Patterns:**

### Product vs Collection
```
Problem:
  /products/running-shoes ‚Üí "running shoes"
  /collections/running-shoes ‚Üí "running shoes"

Solution:
  Collection = broad term ("running shoes")
  Product = specific ("nike air max running shoes")
```

### Blog vs Commercial
```
Problem:
  /blog/best-running-shoes ‚Üí "best running shoes"
  /collections/running-shoes ‚Üí "best running shoes"

Solution:
  Blog = informational ("how to choose running shoes")
  Collection = transactional ("buy running shoes")
```

### Multiple Collections
```
Problem:
  /collections/mens-shoes
  /collections/mens-footwear
  /collections/shoes-for-men
  All targeting "mens shoes"

Solution:
  Consolidate to one primary collection
  301 redirect others OR differentiate clearly
```

**Detection Checklist:**
- [ ] Export all page titles and URLs
- [ ] Look for multiple pages with same target keyword in title
- [ ] Check overlapping H1 tags
- [ ] Search `site:[domain] "[keyword]"` for each target

## 15) Hub-and-Spoke Internal Linking Model

**Structure:**
```
       [COLLECTION PAGE]  ‚Üê Hub
        /     |      \
       /      |       \
   [Product] [Product] [Product]  ‚Üê Spokes
       \      |       /
        \     |      /
         [BLOG POST]  ‚Üê Spoke (links back to hub)
```

**Implementation:**

### Collection as Hub
- Links to top 10-20 products
- Links to related sub-collections
- Links to buying guide content
- Receives links from all spokes

### Products as Spokes
- Link back to parent collection
- Link to 4-6 related products
- Link to relevant guides/blog posts

### Blog Posts as Spokes
- Every post links to relevant products (2-3 minimum)
- Every post links to relevant collection
- Create "Related Articles" section on collection pages

**Cross-Linking Rules:**
```
Collection ‚Üî Collection: 3-5 related category links
Product ‚Üî Product: 4-6 related products ("You might also like")
Blog ‚Üí Product/Collection: Every post links to 2-3+ products
Collection ‚Üí Blog: Add "Related Guides" section
```

## 16) Faceted Navigation Handling

**Problem:** Filters create duplicate URLs
```
/shoes
/shoes?color=red
/shoes?color=red&size=10
/shoes?sort=price
```

**4 Solutions:**

### Option 1: Canonical Tags
All filtered URLs ‚Üí canonical to main collection
```html
<link rel="canonical" href="/shoes" />
```
Best for: Most eCommerce sites

### Option 2: Block in robots.txt
```
Disallow: /*?color=
Disallow: /*?size=
Disallow: /*?sort=
Disallow: /*&
```
Best for: Large sites with crawl budget issues

### Option 3: AJAX Filters (No URL Change)
Filters don't create new URLs ‚Äî JS updates content
Best for: New sites, sites with dev resources

### Option 4: URL Parameter Handling in GSC
Configure which parameters to ignore
Best for: Supplementary to other methods

**Decision Tree:**
```
IF crawl budget is concern ‚Üí robots.txt block
IF already have many indexed filter URLs ‚Üí canonical
IF building new site ‚Üí AJAX filters
ALWAYS ‚Üí configure GSC parameters as backup
```

## 17) Log File Analysis (Crawl Budget)

**Goal:** Optimize how Googlebot spends crawl budget

### Extract Googlebot Data
```bash
# Filter Googlebot requests
grep "Googlebot" access.log > googlebot.log

# Count total requests
grep -c "Googlebot" access.log

# Most crawled URLs
grep "Googlebot" access.log | awk '{print $7}' | sort | uniq -c | sort -rn | head -50

# Find filter parameter waste
grep "Googlebot" access.log | grep "?" | awk '{print $7}' | sort | uniq -c | sort -rn | head -20
```

### Crawl Budget Distribution
```
| Page Type     | % of Crawl | Priority | Status   |
|---------------|------------|----------|----------|
| Products      | 40-50%     | High     | GOOD     |
| Collections   | 20-30%     | High     | GOOD     |
| Homepage      | 5-10%      | Medium   | GOOD     |
| Blog          | 10-15%     | Medium   | GOOD     |
| Filters       | <5%        | Low      | OK       |
| Filters       | >10%       | Low      | WASTE    |
| Search/Cart   | Any        | None     | WASTE    |
```

### Common Crawl Waste (Fix Immediately)
1. **Faceted navigation** ‚Üí Block in robots.txt
2. **Internal search results** ‚Üí `Disallow: /search`
3. **Cart/checkout pages** ‚Üí `Disallow: /cart`
4. **Session ID parameters** ‚Üí Use cookies instead
5. **Sort parameters** ‚Üí Canonical to default

## 18) Quick Reference Commands

```bash
# Fetch robots.txt
curl -s [domain]/robots.txt

# Fetch sitemap
curl -s [domain]/sitemap.xml | head -100

# Check H1 tags (ALWAYS VERIFY BEFORE CLAIMING ISSUES)
curl -s "[url]" | grep -oE '<h1[^>]*>.*?</h1>'

# Check title tag
curl -s "[url]" | grep -oE '<title>.*?</title>'

# Check canonical
curl -s "[url]" | grep -oE '<link[^>]*rel="canonical"[^>]*>'

# Check schema types
curl -s "[url]" | grep -oE '"@type"\s*:\s*"[^"]+"'

# Check meta robots
curl -s "[url]" | grep -oE '<meta[^>]*name="robots"[^>]*>'

# Approximate word count
curl -s "[url]" | sed 's/<[^>]*>//g' | wc -w

# Count internal links
curl -s "[url]" | grep -oE 'href="[^"]*"' | grep -v "http" | wc -l

# Check response headers
curl -sI "[url]" | head -20
```
